## 发布文件

将本目录作为 Git 仓库根目录或发布目录，其中必须包含：

- `index.js`
- `sudoku-core.js`
- `local-ocr.js`
- `package.json`
- `package-lock.json`

启动命令：

```text
npm start
```

运行模板请选择 Node.js `18.x`。图片处理依赖固定为 `sharp@0.33.5`，兼容抖音云 Node.js 18.20.8；请同时上传本包的 `package.json` 和 `package-lock.json`，不要只替换其中一个文件。

服务自动读取抖音云注入的 `_FAAS_RUNTIME_PORT`，不需要硬编码端口。

## 环境变量

```text
SERVER_SALT=至少32位随机字符串
OCR_DAILY_LIMIT=20
```

OCR 使用随服务发布的本地 Tesseract 模型，不使用腾讯云或其他第三方 OCR，不需要任何 OCR 密钥。

## 必须开放的访问路径

```text
/health
/v1/function
/v1/ocr
/v1/ocr-cloud
```

OCR 模型已经包含在 npm 依赖中。运行时只读取抖音云对象存储的临时图片，不需要访问第三方 OCR 服务。

## 接口自测

1. `GET /health`：返回 `{ "ok": true, "service": "magic-number-maze-api" }`。
2. `POST /v1/function`：处理数独反查、题目、成绩和排行。
3. `POST /v1/ocr-cloud`：接收抖音云对象存储临时图片地址，在抖音云服务内执行 OCR。

dev 环境测试通过后再发布 prod。实例数量为 0 是自动缩容状态，首次请求会冷启动；初期可保持最小实例为 0 以节省费用。
